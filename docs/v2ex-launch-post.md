## 正文

最近在做一个 macOS 上的 SSH + SFTP 工具，叫 Remora。  
起因很简单：我自己平时经常在终端、服务器和文件传输之间来回切，现有工具总觉得差一点。要么不像原生 app，要么跑 `vim`、`htop`、`k9s` 这类 TUI 时不够顺手，要么日志一多滚动就开始发飘。后来想想，干脆自己做一个算了。

我现在最想解决的，其实就是这几件事：

1. SSH 和 SFTP 往往是分开的，连上机器之后还得切到别的文件工具。
2. 很多终端工具跑 TUI 应用时，总会在一些细节上别扭。
3. 输出一多、日志一长，滚动性能就容易掉下来。
4. 有些工具功能很全，但在 macOS 上用起来总还是有点不对劲。

所以 Remora 现在的方向也比较直接：

- 把 SSH 会话、SFTP 文件管理、传输流程放到一个界面里，少来回切。
- 把 TUI 兼容性和大输出场景下的滚动性能先做稳。
- 尽量保留 macOS 原生应用该有的感觉。

目前已经在做的点包括：

- 原生 UI，基于 SwiftUI，也有自己的终端引擎
- 针对 `vim`、`top/htop`、全屏终端应用做兼容
- 大输出、长日志场景下尽量保持顺滑
- SSH 和 SFTP 放在同一个工作区里处理

后面准备继续补这些能力：

- 跳板机 / 多级连接
- 导入 `~/.ssh/config`
- 更完整的密钥管理
- 同步/共享连接配置
- AI 支持：这一块已经在做，先从服务器辅助操作、命令解释、日志理解这些方向切入

想先来 V2EX 听听大家的真实需求。  
如果你平时也常用 SSH / SFTP / Terminal，你最希望我优先做的 3 个功能是什么？

比如：

- 跳板机
- 导入 ssh config
- 密钥管理
- 会话同步
- 标签页 / 分栏增强
- AI 辅助操作
- 其他你觉得“没它根本不想换工具”的功能

如果有人愿意试用或者直接拍砖，也欢迎。  
最好是那种“我为什么不会换到这个工具”的真实理由，这类反馈对我最有用。
